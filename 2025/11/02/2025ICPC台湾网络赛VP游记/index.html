2025ICPC台湾网络赛VP游记

2025-11-02T15:28:42.000Z2025-11-02T16:01:20.300Z9593 分钟China

前言

昨天(11月1号)开完第一次大会，今天凌晨就很草率的组好了我们的队伍Terra，因为是三个舟批（目移），然后今天晚上就组织了我们团队的第一次训练赛，基地拉的是今年的台湾网络赛的 vp ，晚上6点到9点。

正赛

因为没有过 ACM 的经验，我们很朴素的从 A 题开始开，AB 都是签到题，然后看 C ，没看懂题目，跳过看了 D 。

这个时候已经过去将近十分钟了，于是我开始分屏写 AB，奇点开 C , A 很顺利的一发通过， B 是一个素数筛然后判断一下就好了，结果 T 了，由于 CF 评测特别慢，看到的时候已经是四十多分钟了，结果发现是数组开的不够大导致在预处理素数的数组的时候越界了，至于为什么是 T 而不是 RE ，我也不是很清楚。

当时奇点没有看懂 C ，青月开了 D ，在打代码，于是我就去看了 C，一对题意发现奇点理解错了，我也把样例推出来了但是没有思路，正着模拟 O(n2) ，会 T。

四十分钟的时候，青月写完 D，交上去等评测。

这个时候刚好发现 B 题错了，于是快速修改了一下，然后就看到了 D 也错了，经过奇点与青月的讨论认为不能简单的枚举判断，应该是一个 dp ，并且推到出了正确的 DP 式，但是交上去在第二个点错了。

在她写 DP 的时候，我发现了 C 可以从后往前做，然后加上判断，就可以做到 O(n) 的时间复杂度，在一旁验证了正确性之后看到她错了立刻换人上机。

然后思路过于抽象把我自己也绕晕了，第二个样例直接没过，经过了长达一个小时的调试终于确认的代码的正确性，交上去发现 T 了，捋了一下代码，在超时了三次之后，终于发现在极端条件下会跑到 O(n2) ，n ∈ (1，2e5)，经过我和队友的一段分析，我突然意识到了在存边的时候前后两条边的关系于是将每次换边的时候的复杂度降到了 O(1) ，整个代码就是 O(n) 的了，顺利通过。

已经是两个小时了，只过了 ABC 三道题，看榜排在铜牌线的位置。

在我打 C 的时候，看到 L 过的人比较多，于是奇点开 L，青月也去看L，两人兴奋的说非常简单并且开始推导，然而还没等我调完代码，他们就改变了他们的看法，但也推导出了正确的式子只是代码会很繁琐。

C 过了之后青月打 L ，四十分钟一直在第二个点错，改到最后甚至样例没过去，遗憾只拿了 ABC 三道题。

最后看分数线，比铜牌线前10名左右，如果能正确写出 DL ，就是银牌。

复盘

直接看题解，然后发现两道题的式子和题解一模一样，非常困惑的调代码但是因为没有 std ， 也没有数据，一直到回寝都没有改出来。

  • 开题开的手忙脚乱。
  • 对英文题的读题能力较差，理解题意需要花费大量时间。
  • 对数组或数据大小的判断力较差，容易开小多一次罚时。
  • 调试能力不足，无法快速有效的查出bug。
  • 不过也体现出青月的推导能力非常的强（Orz）

希望下一次训练赛能过4道题以上叭
